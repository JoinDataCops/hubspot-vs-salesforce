## The migration checklist

1. Audit the database before export. Deduplicate. Flag non-consented records.
2. Map custom fields before migration. Find the format mismatches before they break automations.
3. Validate email addresses at the list level before the migration file exists.
4. Run a 10% test migration. Fix what breaks before touching the full database.
5. Re-collect consent. Don't migrate consent state that can't be demonstrated with a clear audit trail.

## The data layer question

Neither HubSpot nor Salesforce validates data at the source. Bots filling forms become CRM contacts. Duplicate imports stay duplicates. Consent state isn't tracked per contact from the first touchpoint.

The architecture that fixes this sits upstream of both platforms.
